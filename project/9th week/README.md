# Assignment Tasks

## Task 1: Benchmark

### 目标

在 LiveCodeBench 上对齐任意一个模型的评估结果。

### 要求

- 性能差距不超过 **2 个百分点**
- 需要展示技术报告中的指标与性能结果
- 需要展示自己的实验复现结果

### 示例模型

选择模型：

**DeepSeek-R1-Distill-Qwen-7B**

模型链接：

<https://console.volcengine.com/ark/region:ark+cn-beijing/model/detail?Id=deepseek-r1-distill-qwen-7b>

### 参考技术报告

论文标题：

**DeepSeek-R1: Incentivizing Reasoning Capability in LLMs via Reinforcement Learning**

论文链接：

<https://arxiv.org/pdf/2501.12948>

### 汇报内容

需要包括以下内容：

1. 选择的模型名称
2. 模型 API 或调用方式
3. LiveCodeBench 的评估设置
4. 技术报告中的相关指标与性能结果
5. 自己的实验复现结果
6. 技术报告结果与复现结果的对比
7. 性能差距分析

## Task 2: Paper Reading

### 目标

阅读一篇与 **体育视频理解** 相关的 **大模型评估论文**，并以 PPT 的形式进行汇报。

### 汇报要求

- 汇报形式：PPT
- 汇报时间：不超过 **10 分钟**
- 汇报对象：一篇体育视频理解相关的大模型评估论文

### PPT 内容要求

#### 第 1 页：论文基本信息

需要包括：

- 论文标题
- 作者
- 作者单位
- 发表时间
- 发表会议 / 期刊 / 平台（如有）
- 论文链接（如有）

#### 后续内容组织

PPT 后续内容应包括以下部分：

1. **一句话总结**
   - 用一句话概括论文的核心贡献或主要结论。

2. **研究背景和动机**
   - 该研究关注什么问题？
   - 为什么这个问题重要？
   - 现有方法或评估方式存在哪些不足？

3. **研究方法**
   - 论文提出了什么方法、数据集或评估框架？
   - 具体是如何进行体育视频理解评估的？
   - 使用了哪些模型、任务或指标？

4. **实验结果**
   - 主要实验结果是什么？
   - 哪些模型表现较好？
   - 结果说明了什么问题？

5. **价值和局限性**
   - 这篇论文的研究价值是什么？
   - 对体育视频理解或大模型评估有什么启发？
   - 论文还存在哪些不足？

### 拓展建议

根据已读的若干篇论文的研究动机，可以尝试模仿提出一个新的研究动机。

可以从以下角度思考：

- 当前体育视频理解任务中还有哪些场景没有被充分评估？
- 现有大模型在体育视频理解中容易犯哪些错误？
- 是否可以设计一个新的评估维度，例如：
  - 战术理解
  - 动作识别
  - 比赛事件定位
  - 多球员交互理解
  - 裁判判罚理解
  - 长视频时序推理
- 是否可以构建一个更贴近真实比赛分析需求的 benchmark？
